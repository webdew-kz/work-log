# Журнал работ на строительном объекте

## Контекст

Веб-приложение для ведения ежедневного учёта выполненных работ на строительном объекте. Целевой пользователь — прораб, бригадир или ответственный сотрудник, фиксирующий объёмы и виды работ.

## Цель

Создать рабочий monorepo-проект с frontend, backend и базой данных, который запускается одной командой `docker-compose up`. Приложение демонстрирует аккуратную реализацию CRUD с валидацией, REST API, PostgreSQL и удобным интерфейсом.

## Ключевые ограничения

- Нет авторизации
- Один строительный объект
- Без экспорта, аналитики, уведомлений
- Desktop-first веб-интерфейс

## Стек

- Frontend: React, TypeScript, Vite, shadcn/ui, Tailwind CSS, React Hook Form, Zod
- Backend: NestJS, TypeScript, Prisma ORM, PostgreSQL
- DevOps: Docker Compose

## Сущности

### WorkLog (запись журнала)
- date: Date — дата выполнения
- workTypeId: string — связь со справочником
- volume: string — объём с единицей измерения
- executor: string — ФИО исполнителя

### WorkType (вид работ)
- name: string — название вида работ

## Вкладки интерфейса

1. Журнал работ — CRUD, поиск, сортировка, фильтрация
2. Справочник видов работ — CRUD с защитой от удаления используемых записей
